## 0. Why this spec exists

PRD v2.1 has had an initial implementation pass (commit `44471ab`, "PR #6") and several gap-fix PRs (#7–#12). This spec scopes the **closing work** to ship v2.1 to a real-user-usable bar — not a re-design of the feature.

### 1.2 Real gaps to close

1. **G1 — Validation auto-trigger after seeding.** `_seed_org_background` in `app/api/routes/orgs_route.py:78-102` never calls `validate_org`. Newly-onboarded fields stay at `status="seeded"` until a user manually clicks "Verify this".
2. **G2 — Field status maps not serialized.** `OrgProfileResponse` (`app/schemas/__init__.py:470-487`) and the seven sibling entity responses omit `field_status_map`, `field_confidence_map`, `field_source_map`. Frontend reads these (`CompanyProfile.tsx:174`) but always sees `undefined` → all ticks render as empty.
3. **G3 — Test suite missing.** Zero PRD test cases implemented. `aegis-backend/requirements.txt` has no pytest. No Playwright. No CI workflows.
4. **G4 — TC-U-* / TC-A-* coverage missing** for behaviour that already works (session reset, backend permissions). Tests pin the existing correct behaviour against regressions.

### PR 2 — Backend correctness fixes (G1 + G2)

**G1 — validation auto-trigger.** Edit `_seed_org_background`:

```python
# app/api/routes/orgs_route.py
async def _seed_org_background(org_id: UUID):
    try:
        await seed_org_from_fingerprint(org_id)
        await seed_org(org_id)
        from app.validation.orchestrator import validate_org
        await validate_org(org_id)
    except Exception:
        logger.exception("seed+validate failed", org_id=org_id)
```

**Resilience caveat:** if the FastAPI process restarts mid-validation, fields stuck at `status="seeded"` are not re-validated by the daily reseed task (it handles `unknown` only). Verify in implementation that `validate_org` catches per-field failures internally so only the failing field stays at `seeded`. If it does not, add per-field try/except inside the orchestrator.

**G2 — schema additions.** Add three optional dict fields to each response schema:

```python
class OrgProfileResponse(BaseModel):
    # ... existing fields ...
    field_status_map: dict[str, str] = Field(default_factory=dict)
    field_confidence_map: dict[str, float] = Field(default_factory=dict)
    field_source_map: dict[str, list[str]] = Field(default_factory=dict)
    class Config:
        from_attributes = True
```

Apply identically to: `OrgProfileResponse`, `LOBResponse`, `GeographyResponse`, `OrgIndustryResponse`, `ProductResponse`, `CustomerSegmentResponse`, `ThirdPartyDependencyResponse`, `DataTechnologyProfileResponse`.

### 5.2 Mock-Claude harness contract

`MockClaudeBank` raises `AssertionError` on any unknown `(agent, company, field)` triple. This is deliberate — silent fallback to a default response is the most common source of false-green tests in LLM systems. Fixtures must be explicit.

Recording new fixtures: developer runs `LIVE_CLAUDE=1 python tests/scripts/record_fixtures.py --company uber --field legal_name`. The script invokes the relevant agent against real Claude, captures the response, writes JSON.

## 6. Risk register

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| `validate_org` lacks per-field error handling, so one Claude failure kills the whole batch | Medium | Medium | PR 2: verify and add try/except per field in orchestrator if needed |
| FastAPI process restarts mid-validation, stranding fields at `seeded` | Low | Low | Accept short-term; users can click "Verify this". Optional follow-up: hourly "stragglers" sweep task. |
| Test-Postgres differs from prod-Postgres on JSONB defaults | Low | Low | Use Postgres 16 in CI services to match Railway. |
| Playwright tests flake on real timing | Medium | Medium | Use explicit `await expect(...).toHaveAttribute` with default 5s timeout; avoid `waitForTimeout`. |
| Recorded fixtures drift from current prompts/models | High over time | Medium | `record_fixtures.py` re-runs are explicit; nightly live-Claude smoke catches drift; PRD prompt changes must include fixture refresh in same PR. |
| Adding `field_status_map` to response breaks frontend types | Low | Low | Field is additive; frontend already reads it opportunistically. |
| Mock harness hides a real bug in agent code | Medium | Medium | Nightly live-Claude job runs the 5 smoke tests enumerated in §4 PR 3 against real Claude; alerting deferred (PagerDuty out of scope) but failures visible in GitHub Actions log + email-on-fail. |
